# Task 3.3 - Expo Moderator App Core

## Summary

Implementei o primeiro protótipo Expo/React Native do app de moderadores em `apps/moderator`, com ativação por convite, persistência de sessão, contexto do evento, lista de fotos pendentes, revisão e ações de aprovar/rejeitar. A branch foi commitada com `4189834 Add moderator app prototype`.

## Details

Li a arquitetura em `docs/ARQUITETURA_APP_MODERADOR.md`, o contrato em `docs/API_APP_MODERADOR.md` e as rotas backend de `/api/moderator-app/*`. Em seguida criei um pacote Expo separado dentro do workspace para manter a implementação nativa isolada do app Next.js principal.

O app implementa:

- configuração de backend por `EXPO_PUBLIC_MODERATOR_API_BASE_URL` e campo editável na tela de login;
- ativação de sessão via `POST /api/moderator-app/sessions`;
- persistência de token de sessão, device id e URL de backend com `expo-secure-store` em iOS/Android e fallback em memória no web;
- restauração de sessão com `GET /api/moderator-app/me`;
- listagem de pendentes com `GET /api/moderator-app/photos?status=PENDING`;
- UI de revisão de foto com imagem, nome, mensagem e horário;
- decisão de aprovar/rejeitar via `POST /api/moderator-app/photos/{photoId}/decision`;
- logout via `DELETE /api/moderator-app/sessions/current`;
- estados de loading, vazio, erro, refresh e confirmação de rejeição;
- README com instruções de instalação, execução, URL local e limitações.

O app não implementa upload de convidados e mantém a moderação web como fallback.

Durante a validação, o typecheck raiz passou a enxergar tipos React Native porque `tsconfig.json` incluía `**/*.ts` e `**/*.tsx`. Corrigi isolando `apps` do programa TypeScript raiz; o pacote Expo mantém seu próprio `tsconfig.json` e typecheck separado.

## Compatibility Concerns

Adicionar React Native dentro do monorepo introduz tipos globais incompatíveis com o programa TypeScript do Next.js se o app Expo for incluído pelo `tsconfig.json` raiz. Resolvi excluindo `apps` do typecheck raiz e mantendo validação separada para `@eventoon/moderator-app`.

## Important Findings

O pacote Expo precisa permanecer validado por filtro/package próprio. Com RTK, `rtk pnpm --filter ... typecheck` reescreveu o comando e rodou o typecheck raiz; para este comando específico foi necessário usar `rtk proxy pnpm --filter @eventoon/moderator-app typecheck`.
