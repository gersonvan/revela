# Task 3.2 - Moderator App Backend

## Summary

Added prototype backend support for the moderator native app while preserving the existing web moderator token/cookie flow. The app now has session activation, Bearer session authentication, event-scoped context, photo list/detail, approve/reject decision, push-token persistence, and logout endpoints.

## Details

Read `docs/ARQUITETURA_APP_MODERADOR.md` and implemented the minimal backend contract proposed there. Added `ModeratorSession` to Prisma so app sessions are revocable and tied to existing `Moderator` records; every app API derives `eventId` from the authenticated moderator session instead of trusting client-provided event IDs.

Created helpers under `src/lib/moderation` for app session authentication and transacional photo decisions. The decision helper updates `Photo` and creates `ModerationDecision` together, using the same status/action mapping as the current web moderation flow.

Implemented JSON routes under `/api/moderator-app`: session activation, current-session logout, `/me`, photo list, photo detail, photo decision, and push-token registration. Push support is intentionally no-cost and persistence-only in this phase; no push dispatch is attempted.

Added `docs/API_APP_MODERADOR.md` with endpoint behavior and caveats for the Expo app implementation. Added `scripts/smoke-moderator-app-api.mjs` and `pnpm smoke:moderator-app` for local end-to-end validation when the dev server and Postgres are available.
