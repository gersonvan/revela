# Task 3.4 - Push Notification Prototype

## Summary

Implementei o registro app-side de push no protótipo Expo do moderador, integrado ao endpoint backend existente `PUT /api/moderator-app/push-token`. A branch foi commitada com `0402581 Add moderator push registration`.

## Details

Trabalhei somente em `apps/moderator`, mantendo upload de convidados fora do escopo e sem implementar dispatch backend de push. O app agora usa `expo-notifications` e `expo-device` para solicitar permissão, gerar Expo Push Token quando o ambiente suporta e registrar o token na sessão autenticada do moderador.

O fluxo implementado:

- moderador entra normalmente com convite e sessão existente;
- área "Alertas de fotos pendentes" aparece quando `/me` indica `canRegisterPushToken`;
- ao tocar em `Ativar`, o app solicita permissão local;
- web, simulador/emulador, permissão negada ou erro de credenciais Expo retornam mensagem clara sem bloquear moderação;
- quando há token Expo válido, o app chama `PUT /api/moderator-app/push-token` com `pushToken`, `platform` e `appVersion`;
- falha de push não bloqueia lista pendente, aprovar/rejeitar, logout nem fallback web.

Também documentei o comportamento planejado para alertas agrupados: janelas curtas por evento, throttling em pico, mensagem genérica sem foto/nome/mensagem/token e abertura da lista pendente.
