# Task 3.1 - App Architecture API Contract

## Summary

Produzi a documentação técnica do app moderador em Expo/React Native, mantendo o app nativo restrito a moderadores e preservando o fallback web de moderação. O documento cobre fluxo de dados, telas, contrato de API, registro de sessão/dispositivo, notificações push, distribuição e restrições de segurança.

## Details

Inspecionei o fluxo atual de moderação web e escopo por evento em `src/lib/moderation`, `src/app/moderate/[token]/page.tsx`, `src/app/moderate/[token]/actions.ts`, `src/components/moderation/bulk-moderation-actions.tsx`, `prisma/schema.prisma`, `src/app/admin/events/actions.ts`, `src/app/admin/events/[eventId]/page.tsx`, `src/app/api/events/[slug]/photos/route.ts` e `docs/QA-2026-07-05-validacao-fluxo-web.md`.

Com base nessa inspeção, documentei uma arquitetura de app nativo que reaproveita o modelo atual de moderadores por evento, token secreto com hash, revogação e histórico de decisões, mas propõe uma camada futura de sessão de app revogável para não repetir o token de convite em cada chamada.

Nenhuma rota backend nem código Expo foi implementado nesta Task, conforme instrução.

## Important Findings

O modelo atual já suporta convite de moderador por evento, token com hash, cookie de dispositivo no web e revogação, mas ainda não possui uma entidade de sessão nativa para app com token próprio, push token, expiração e metadados do dispositivo. O documento trata essa camada como proposta futura e não como comportamento implementado.
