# Task 2.2 - Upload And History Behavior

## Summary

Applied event moderation mode to guest uploads and preserved automatic approval history. Improved the guest photo selector limit handling so exceeding 15 selected images keeps any existing valid selection and shows a clear Portuguese warning.

## Details

Updated the upload API to read `Event.moderationMode`. Events with `WITH_MODERATION` keep the existing behavior and create photos as `PENDING`; events with `WITHOUT_MODERATION` create photos as `APPROVED` so they appear in the approved-photo feed.

Added `AUTO_APPROVED` to `ModerationAction` and made `ModerationDecision.moderatorId` nullable so automatic decisions can be recorded without pretending a human moderator approved the photo. The existing moderation actions still operate on approved photos, so moderators can reject/remove auto-approved photos later.

Added `scripts/smoke-moderation-mode.mjs` and `pnpm smoke:moderation-mode` to verify pending vs auto-approved upload behavior, automatic history, and approved feed inclusion when a local server and database are available.
