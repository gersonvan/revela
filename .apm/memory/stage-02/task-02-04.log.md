## Important Findings
- The branch includes `EventModerationMode` with default `WITH_MODERATION`.
- `POST /api/events/[slug]/photos` creates `PENDING` photos for `WITH_MODERATION` and `APPROVED` photos for `WITHOUT_MODERATION`.
- Auto-approved uploads create `ModerationDecision` with `AUTO_APPROVED`, `previousStatus: PENDING`, `newStatus: APPROVED`, and no human `moderatorId`.
- The selection-limit client logic preserves an existing valid selection when a later selection exceeds 15 files and shows a Portuguese message explaining that the prior selection was kept.
- Approved photos remain removable from the moderation UI via the existing approved-photo rejection/removal path.
