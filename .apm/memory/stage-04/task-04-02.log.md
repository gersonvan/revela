# Task 4.2 - Controlled Video Processing Spike

## Summary
Built a local/dev-only video proof that converts a sample video into a 5-10 second MP4 clip, poster image, and JSON manifest without touching production upload, moderation, screen, export, schema, or storage behavior.

## Details
Integrated the prior architecture study from `docs/ESTUDO_VIDEO_BAIXO_CUSTO.md` and selected the safest proof mechanism: a local Node script that shells out to `ffmpeg`/`ffprobe`. The script accepts a local sample file, validates the requested duration window, writes generated artifacts under ignored `storage/video-proof/output/`, strips audio by default, and records metadata plus the original-retention note in a manifest.

Added an operator-facing runbook documenting prerequisites, input/output paths, command examples, output format, constraints, failure modes, security/privacy/cost implications, and validation evidence.

## Issues
`pnpm video:proof` could not be used in this worktree because `node_modules` is not installed. The script has no external package dependency, so validation used direct `node scripts/video-proof.mjs ...`; the runbook documents this fallback.

## Important Findings
The local proof works without cloud infrastructure and can produce a short clip well below the target 3 MB size for a synthetic sample. Before treating the settings as product-ready, the same script should be tested with real cellphone videos because compression ratio, perceived quality, rotation metadata, and processing time can vary materially by device and codec.
